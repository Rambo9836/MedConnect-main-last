# MedConnect Non-PHI Pilot Policy

This launch is a **non-PHI pilot**. The goal is to validate product usability and reliability with real users while avoiding regulated data handling risk.

## What is allowed
- Basic account profile details (email, first name, last name)
- Community interactions and non-sensitive study engagement data
- General interest/condition tags without direct identifiers

## What is NOT allowed
- Uploading raw medical documents containing patient identifiers
- National identifiers, insurance IDs, hospital MRNs, or lab report PDFs with names
- Clinical notes that can identify a person directly

## User responsibilities
- Users must not submit direct patient identifiers or regulated clinical records.
- Users should use anonymized or synthetic examples when testing flows.

## Operator responsibilities
- Promptly remove disallowed submissions if reported.
- Keep audit trail of moderation/removal actions.
- Communicate clearly that this pilot is not a certified medical data platform.

## Escalation path
- For any suspected sensitive data submission:
  1. Restrict access to the content immediately
  2. Remove content after review
  3. Document incident and mitigation
  4. Notify impacted user if required
